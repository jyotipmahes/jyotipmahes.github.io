Leanplum in App Purchase Prediction

Predicting the probability of a user making a purchase in next 7 and 14 days

Objective

This project was a part of a in-classroom kaggle competition. The objective of the competition was to make prediction on whether an user will make an in-app purchase or not in next 7 and 14 days. Lean Plum was kind enough to share the data for the competition.

Approach and Results
The dataset consisted of 50gb+ data relating to user engagement, session level events, user level app specific attributes and message triggered data. We created 100+ features on each user based on various time frames. We treat the data as time series and create various central statistics as features relating to user engagement. We create data labels by taking last 2 week data and assign labels to each user based on purchase history. For modelling, we started by fitting models to individual data sets. Next, we selected best features using Boosting and Random Forest model and merged the whole dataset to improve performance. Finally, we used ensembling to further improve model performance. The competition used AUC as the evaluation metric and the best model achieved an AUC of 0.975.

  • Tools used: Python(pandas, numpy, scikit-learn, matplotlib)
  • Techniques: Feature engineering, Random Forest, Xgboost, Ensembling

LANL Earthquake Prediction

Predict the time remaining before laboratory earthquakes occur

Objective

This project is based on LANL Earthquake Prediction Challenge dataset. The main objective of the project was to create a distributed database and spark machine learning pipeline on a big dataset. In terms of modelling and prediction, we were trying to predict the time left for the next laboratory Earthquake.

Approach and Results
Storing on S3: We started by storing the whole data on a S3 bucket.
Feature Extracting using EC2 : The raw data is a time series data. Our objective was to look at a sequence of 150000 data points and predict the time for the next Earthquake from the last time step. We decided to extract a lot of time series features. Since the data is large to fit on a local system, we use AWS EC2 instance for feature engineering. We used c5d.4xlarge instance to created 700K featured datapoints.
MongoDB : We store the featured data by creating a MongoDB database. We created a shared and distributed database consisting of 2 shards and 3 nodes per shard. We also created 3 node server and 2 configuration server(a total 0f 11 servers).
Modelling : We use AWS EMR clusters at different configuration to build Spark ML models. In our experiment, we found that Random Forest was the best performing algorithm. For that reason, we decided to run cross-validation to further improve our results. Our lowest RMSE score is 1.40, which is a significant improvement from our previous score of 2.20 for Random Forest with default parameters.

  • Tools used: Python(pandas, numpy, scikit-learn, matplotlib), Spark ML, AWS EC2, AWS EMR, MongoDB, AWS S3
  • Techniques: Feature engineering, Linear Regression, Decision Tree, Random Forest, Regression, Sharding
  • Github: Code
  • Report: Report

Isolation Forest for Anomaly detection

From scratch implementation of Isolation forest algorithm for Anomaly detection and modifications to be make it robust to noisy features.

Objective

The goal of this project is to implement the original Isolation Forest algorithm by Fei Tony Liu, Kai Ming Ting, and Zhi-Hua Zhou as a part of MSDS689 course. I also try to modify the original algorithm to make it robust to noisy features.

Approach
The algorithm is based on the idea that the anomalies occur in isolation and it is much easier to separate anomalies using random split in the feature range compared to normal instances. To simplify, anomalies are rare and different which occurs in isolations. The algorithm randomly selects a small subset of observations. Next, we randomly select a feature column and a split value from the selected feature which lies between the min and max values. Note that, the current algorithm only works for continuous features. We keep growing the trees till the termination condition is reached. We repeat the same process and grow a forest to reduce variance. Anomaly score is calculated based on the average path length of each observation across the forest. We normalize the length with the standard height of a binary tree and expect the length of the anomaly to be smaller compared to normal instances. The improved version changes the logic of selecting the split value such that the splits are on the extremes rather than selecting from uniform distribution.

  • Tools used: Python(pandas, numpy, scikit-learn, matplotlib)
  • Techniques: Tree implementation, Anomaly detection, Unsupervised learning
  • Github: Code

Article Recommendation

Article recommendation engine based on article similarity built using Flask

Objective

The project was a part of MSDS692 course and the goal of this project was to learn how to make a simple article recommendation engine using a semi-recent advance in natural language processing called word2vec (or just word vectors). In particular, I have used Stanford's GloVe project trained on a dump of Wikipedia for the word embeddings.

Approach
I used the pre-trained GloVe embeddings to find the document centroid for each document. Once I had the centroids, i found the distance between different documents based on the centroids and selected the documents nearest to a given document as recommendation. I also built a web server using Flask to serve this model.

  • Tools used: Python(pandas, numpy, scikit-learn), AWS EC2, Flask, Gunicorn
  • Techniques: NLP, Tokenization and Cleaning, NN Search, Hashtable
